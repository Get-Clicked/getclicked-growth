# Notion Style Guide

Shared formatting, voice, and block primitives for all getClicked plugin skills.
Every Notion page we write follows these rules. Skill-specific templates live in each SKILL.md.

---

## Page Anatomy

Every page follows this top-to-bottom structure:

1. **Status badge** -- first line, always a quote block:
   ```
   > **Status: Draft** | Generated by /skill on YYYY-MM-DD
   ```
   Two statuses: `Draft` (skill just wrote it), `Final` (client approved).

2. **Executive summary** -- 2-3 plain prose sentences. What is this, the headline finding, what to do about it. This is the "screenshot to Slack" paragraph.

3. **Narrative lead** -- opens with story/problem/context, not a bullet dump. Sets the "why this matters" frame.

4. **Body sections** -- H2 for major sections, H3 for subsections. Never H4+.

5. **Horizontal rules** (`---`) between major H2 sections as visual breaks.

6. **Source citations** -- every section using data ends with:
   ```
   > Source: DataForSEO, pulled YYYY-MM-DD
   ```

---

## Formatting Rules

- **Bold** for emphasis within prose -- not for structure.
- *Italics* for positioning context, strategic rationale, framing (*"The market is shifting toward..."*).
- Toggle sections via `<details>` tags for supporting data behind a narrative point -- keeps pages scannable.
- Tables only for genuinely tabular data (keywords, competitors, budgets) -- never as layout.
- Bullet lists for discrete items. Numbered lists only for sequences/steps.
- No emoji. No colored text. No icon prefixes.
- H2/H3 only. No H4+. No H1 (Notion page title is H1).

### Toggle syntax

```markdown
<details>
<summary><strong>Supporting data: Market size by region</strong></summary>

[Table or detailed content here]

</details>
```

## Database Conventions

- Every database row gets a `Status` select property: `Draft` / `Final`.
- `Source` column on every database: which skill + date wrote the row.
- Number formatting: Volume as plain number, CPC as `$X.XX`, percentages as `XX%`.

## Voice and Tone

Write like the client's sharpest colleague -- someone who enjoys the work and makes them smarter.

### Rules

- **Have a point of view.** Don't hedge with "consider" and "it may be worth exploring." Say what you think and why.
- **Write like you talk.** Short sentences. Occasional fragments. The kind of thing you'd say in a meeting when you're excited about the data.
- **Be specific, not comprehensive.** "Idaho CPCs are 40% cheaper than Colorado -- that's your beachhead" beats "Consider geo-targeting strategies to optimize cost efficiency across markets."
- **Use the client's language.** If they say "paddles" don't say "products."
- **Surprise with insight.** Every section should have at least one thing the client didn't already know.
- **Kill the filler.** No "in today's competitive landscape." No "it's important to note." No "leveraging synergies."
- **Earn the toggle.** The narrative above the fold should be interesting enough that the reader *wants* to expand the details.
- **The Honest Take.** Every deliverable has a moment where it stops reporting and starts advising. Name the trade-offs. Say what's not working.

### Anti-patterns (never do these)

- "This comprehensive analysis reveals..."
- "Based on our research, we recommend..."
- "Key takeaways include..."
- "The following table shows..."
- Explaining what a section is before the section says it
- Restating headers as sentences

### The test

If Steph screenshots a section and sends it to her CEO in Slack, does it make her look smart? If yes, ship it. If it reads like a template, rewrite it.

## Block Primitives

Nine reusable building blocks for all skills.

### 1. Status Badge

```markdown
> **Status: Draft** | Generated by /brand on 2026-03-19
```

### 2. Executive Summary

Plain prose, 2-3 sentences. No headers, no bullets. The "what is this and why should I care" paragraph.

### 3. Key Finding Callout

```markdown
> **Key Finding:** Idaho CPCs run 30-50% cheaper than Colorado for the same keywords.
> Geo-targeting these markets first maximizes budget efficiency.
```

### 4. Data Table with Source

```markdown
| Keyword | Volume | CPC | Competition |
|---------|--------|-----|-------------|
| pickleball paddles | 49,500 | $1.82 | High |

> Source: DataForSEO, pulled 2026-03-19
```

### 5. Toggle/Detail Section

```markdown
<details>
<summary><strong>Supporting data: Market size by region</strong></summary>

[Table or detailed content here]

</details>
```

### 6. Recommendation Block

```markdown
### Recommended next step

[Specific action with numbers and timeline.]

> *Based on /ads keyword data and /optimize Week 1 performance.*
```

### 7. Competitor Entry

```markdown
- **CompetitorName** -- [1-2 sentences: what they do well, where they're weak, what it means for us]
```

### 8. Cross-Skill Reference

```markdown
> See also: Ad Groups (Ads section) | Landing Pages (Landing section)
```

Use plain text references. During Notion sync, the agent resolves these to @-mentions if page IDs are known.

### 9. The Honest Take

```markdown
## The honest take

[Stop reporting, start advising. Name the trade-offs. Say what's uncomfortable.
Every deliverable MUST include this moment.]
```

Can be titled "The honest take," "The opportunity," "The big bet," or "What this means for you" -- the pattern is the same: stop presenting data, start giving advice.
